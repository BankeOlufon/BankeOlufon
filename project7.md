# 🔒 Linux Web App Exploitation Lab (Initial Access)

## 📌 Project Overview
An offensive security lab simulating the **initial access phase** of penetration testing against a vulnerable Linux web application.  
This project demonstrates how attackers exploit misconfigured web apps to gain shell access, forming the foundation of OSCP-style workflows.

---

## 🎯 Core Areas Covered
- Web Application Vulnerability Discovery  
- Exploitation & Shell Access on Linux Targets  
- Enumeration & Post-Exploitation Basics  
- OSCP Initial Access Methodology  

---

## 🛠️ Technical Tasks & Functions Performed

### Vulnerable Web Application Setup
- Deployed an intentionally vulnerable Linux machine hosting a broken web app (e.g., fake corporate blog).  
- Configured the environment to simulate real-world corporate exposure.  

### Exploitation Workflow
- Used **Kali Linux tools** to scan and enumerate the web application.  
- Identified exploitable flaws (e.g., file upload vulnerability).  
- Uploaded malicious payloads to gain command execution.  
- Forced the server to grant a basic shell inside the Linux system.  

### Validation & Analysis
- Verified shell access by executing system commands.  
- Documented exploitation steps and confirmed persistence.  
- Connected findings to OSCP methodology: initial foothold before privilege escalation.
